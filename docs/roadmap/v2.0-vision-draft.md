**Date:** 2026-05-16
**Status:** DRAFT — Pre-planning
**From:** v1.9.0-stable
**Target:** v2.0.0-stable (Q1 2027)

---

## Executive Summary

v2.0 represents the evolutionary next phase of ed2kIA: transitioning from a research-grade distributed interpretability network to a production-ready platform with full GUI (Desktop/Mobile), enterprise-grade ZKP infrastructure, decentralized governance, and enterprise integrations.

**Core Themes:**
1. **Full GUI** — Desktop (Tauri) + Mobile (React Native) + Browser Extension
2. **ZKP v2** — Native proof aggregation, multi-curve support, formal verification
3. **Decentralized Governance v2** — On-chain proposals, reputation-weighted voting, DAO transition
4. **Enterprise** — K8s operator, Prometheus exporters, SSO integration

## 3. ZKP v2 — Enterprise-Grade Proofs

### 3.1 Native Proof Aggregation

**Current:** Proof aggregation in application layer (proof_aggregation.rs)
**Target:** Native aggregation in ZKP circuit layer

**Features:**
- Batch proof generation with native circuit support
- Proof composition (proofs of proofs)
- Recursive SNARK support
- Multi-party computation (MPC) integration

### 3.2 Multi-Curve Support

**Current:** BN254 only (ark-bn254)
**Target:** Multi-curve with automatic selection

| Curve | Use Case | Performance |
|-------|----------|-------------|
| BN254 | General purpose | Baseline |
| BLS12-381 | Signature aggregation | +15% proof size |
| Bls12-377 | Enterprise compliance | -10% proof time |
| Pasta (Mina) | Recursive proofs | Native recursion |

### 3.3 Formal Verification

**Scope:**
- Kani verification for critical ZKP circuits
- CBMC for memory safety in proof generation
- Prover/Verifier correctness proofs

**Timeline:** Q1 2027

## 4. Decentralized Governance v2

### 4.1 On-Chain Proposals

**Current:** GitHub-based proposals with Ed25519 signatures
**Target:** On-chain governance with smart contract execution

**Features:**
- Proposal submission via smart contract
- Reputation-weighted voting (on-chain)
- Time-locked execution
- Quorum requirements (≥30% participation)

### 4.2 Reputation-Weighted Voting

**Mechanism:**
- Each node's voting power = f(reputation, stake, uptime)
- Reputation decay prevents vote buying
- Sybil resistance through proof-of-contribution

### 4.3 DAO Transition

**Phases:**
1. **Hybrid (v2.0):** GitHub + on-chain proposals
2. **Majority On-Chain (v2.1):** ≥50% proposals on-chain
3. **Full DAO (v2.2):** Complete decentralized governance

## 5. Enterprise Integrations

### 5.1 Kubernetes Operator

**Scope:**
- Helm chart for ed2kIA node deployment
- Custom Resource Definition (CRD) for node management
- Auto-scaling based on network load
- Health checks and self-healing

**Deliverables:**
- `ed2kia-operator/` repository
- Helm chart `ed2kia-node`
- Documentation for K8s deployment

### 5.2 Prometheus Exporters

**Scope:**
- Full metrics export for all subsystems
- Custom metrics for ZKP, SAE, P2P, governance
- Grafana dashboards (pre-built)
- Alerting rules

**Key Metrics:**
```

ed2kia_sae_inference_time_ms{model="qwen-scope", dim=8192}
ed2kia_p2p_peer_count{region="us-east"}
ed2kia_governance_active_proposals
ed2kia_reputation_avg_score
```

### 5.3 SSO Integration

**Scope:**
- OAuth 2.0 / OIDC support
- SAML for enterprise deployments
- LDAP/Active Directory integration
- Role-based access control (RBAC)
