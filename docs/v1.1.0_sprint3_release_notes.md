## 📋 Resumen

Sprint 3 introduce dos componentes críticos para la escalabilidad y descentralización de ed2kIA:

1. **Motor ZKP Asíncrono** — Generación de pruebas de conocimiento cero con fallback Merkle+VRF, verificación en pool de 4 workers, y acumulación de batches para reducción de costo de verificación.
2. **Marketplace Descentralizado de Recursos v2** — Matching SLO-aware de shards SAE, VRAM y ancho de banda con escrow criptográfico (redb + ed25519-dalek), pricing adaptativo con compromisos Pedersen, y anti-monopolio.
3. **Puente ZKP↔Marketplace** — Pipeline unificado que orquesta P2P → Matching → ZKP → Escrow → Liberación → Consenso.

## 🔒 Seguridad

### Criptografía
- **Curva:** BN254 (ark-bn254) con ark-ec/ark-ff
- **Compromisos:** Pedersen para precios y batches
- **Firmas:** ed25519-dalek (ed25519) para escrow y auditoría
- **Hash:** SHA-256 para batches y integridad

### Escrow
- Base de datos: `redb` (inmutable, transaccional)
- Estados: `Locked → Delivered → ZKPVerified → Released`
- Refund: por SLO no cumplido o disputa
- Disputa: requiere intervención de gobernanza

### Anti-Monopolio
- Límite: 30% de recursos de un tipo por nodo
- Fallback automático al siguiente mejor listing
- Transparencia total en algoritmo

## 🔄 Guía de Migración

### De v1.0.0-stable a v1.1.0-sprint3

1. Asegurar que `Cargo.toml` incluye `"v1.1-sprint3" = []` en `[features]`
2. Añadir feature flag en `Cargo.toml` para tests:
   ```toml
   [[test]]
   name = "v1_1_sprint3_e2e"
   path = "tests/integration/v1_1_sprint3_e2e.rs"
   required-features = ["v1.1-sprint3"]
   ```
3. En `src/lib.rs`, añadir módulos:
   ```rust
   #[cfg(feature = "v1.1-sprint3")]
   pub mod marketplace_v2 { ... }
   
   #[cfg(feature = "v1.1-sprint3")]
   pub mod bridge_v2 { ... }
   ```
4. Ejecutar: `cargo check --features v1.1-sprint3`

### Uso del Bridge

```rust
use ed2kia::bridge_v2::zkp_marketplace_bridge::ZKPMarketplaceBridge;
use ed2kia::marketplace_v2::matchmaker::{ResourceListing, ResourceRequest, ResourceType};

let mut bridge = ZKPMarketplaceBridge::new("/tmp/escrow.db")?;

// Publicar recurso
bridge.publish_resource(ResourceListing {
    node_id: "my_node".into(),
    resource_type: ResourceType::SAEShard {
        model_id: "scope-v2".into(),
        layer: 5,
    },
    quantity: 10.0,
    base_price: 100.0,
    listed_at: chrono::Utc::now().timestamp_millis() as u64,
    expires_at: (chrono::Utc::now().timestamp_millis() as u64) + 3600000,
    max_latency_ms: 100,
    availability_slo: 0.99,
    min_throughput: 1000,
});

// Ejecutar workflow
let result = bridge.execute_resource_workflow(ResourceRequest {
    requester_id: "buyer".into(),
    resource_type: ResourceType::SAEShard {
        model_id: "scope-v2".into(),
        layer: 5,
    },
    quantity: 5.0,
    max_price: 150.0,
    max_latency_ms: 100,
    min_availability: 0.95,
}).await?;

if result.success {
    println!("Workflow completed: {:?}", result.final_state);
}
```
