## 2. Dashboard v2 — Motor de Estado

### 2.1 Estructura Principal

```rust
    last_request_ms: u64,
}
```

### 2.2 Ventanas Deslizantes

Las ventanas deslizantes (`SlidingWindow`) mantienen un historial de métricas dentro de una ventana de tiempo configurable:

- **window_ms**: Duración de la ventana en milisegundos (default: 60_000ms = 60s)
- **max_size**: Máximo de puntos de datos (default: 10_000)
- **cleanup**: Limpieza automática de puntos expirados

### 2.3 Sistema de Alertas

Las alertas se generan automáticamente cuando las métricas superan umbrales configurables:

| Métrica | Warning | Critical |
|---------|---------|----------|
| `SystemCpuUsage` | > 76.5% | > 85% |
| `SystemMemoryUsage` | > 81% | > 90% |
| `SystemNetworkLatency` | — | > 200ms |
| `AlignmentConfidence` | < 0.6 | < 0.5 |
| `FederationTrustScore` | < 0.36 | < 0.3 |

### 2.4 Rate Limiting

Protección contra abuso con límite de requests por segundo configurable (default: 10 req/s).

### 4.4 Circuit Breaker

- **Umbral**: 5 fallos consecutivos (configurable)
- **Recovery**: Semi-abierto después de health_check_interval_ms
- **Exclusión**: Nodos con circuit breaker abierto son excluidos del routing

### 4.5 Puntuación Compuesta

```
score = latency_weight * (1 / avg_latency)
      + reputation_weight * reputation
      + slo_weight * slo_compliance
```

### 5.2 Regresión Lineal

El balancer usa regresión lineal simple para predecir tendencias:

```
y = mx + b
m = (n*Σxy - Σx*Σy) / (n*Σx² - (Σx)²)
b = (Σy - m*Σx) / n
```

## 6. Integración entre Módulos

### 6.1 Dashboard → WS Stream

```
DashboardState.get_snapshot()
    └─▶ DashboardSnapshot { metrics, nodes, alerts, summary }
        └─▶ serde_json::to_value(&snapshot)
            └─▶ WsDashboardStream.broadcast_snapshot(json_data)
                └─▶ DashboardMessage::Snapshot { sequence, timestamp_ms, data }
```

### 6.2 Router → Balancer

```
AdaptiveRouter.route(model_type, predictive_score)
    └─▶ If enable_predictive AND confidence >= threshold:
            └─▶ PredictiveBalancer.compute_node_score(node_id)
                └─▶ RoutingDecision { target_node, strategy: Predictive, ... }
```

### 6.3 Full Pipeline

```
┌─────────────┐     ┌──────────────┐     ┌──────────────┐     ┌─────────────┐
│  Metrics    │     │  Dashboard   │     │ WS Stream    │     │  Client     │
│  Ingested   │────▶│  State       │────▶│  Broadcast   │────▶│  Browser    │
│             │     │  Engine      │     │              │     │             │
└─────────────┘     └──────┬───────┘     └──────────────┘     └─────────────┘
                           │
                           ▼
                    ┌──────────────┐     ┌──────────────┐     ┌─────────────┐
                    │ Adaptive     │────▶│ Predictive   │────▶│ Node        │
                    │ Router       │     │ Balancer     │     │ Selection   │
                    │              │     │              │     │             │
                    └──────────────┘     └──────────────┘     └─────────────┘
```

## 8. Patrones de Diseño

| Patrón | Módulo | Implementación |
|--------|--------|----------------|
| **State Machine** | Dashboard v2 | Metric recording → Alert generation → Snapshot |
| **Circuit Breaker** | Adaptive Router | Open → Half-Open → Closed states |
| **Sliding Window** | Dashboard v2 | Ventanas temporales con cleanup automático |
| **Strategy** | Adaptive Router | Adaptive/Predictive/Fallback routing |
| **Observer** | WS Stream | Broadcast a conexiones suscritas |
| **Composite Score** | Router + Balancer | Ponderación de múltiples métricas |

## 9. Consideraciones de Rendimiento

| Operación | Complejidad | Target |
|-----------|-------------|--------|
| `record_metric` | O(1) amortizado | < 1ms |
| `get_snapshot` | O(n) metrics + O(m) nodes | < 5ms |
| `route` | O(k) candidates | < 1ms |
| `predict_metric` | O(h) history | < 2ms |
| `broadcast_snapshot` | O(c) connections | < 10ms |

## 10. Seguridad

- **Rate Limiting**: Todas las conexiones WebSocket tienen rate limiting por defecto
- **Autenticación**: Conexiones deben autenticarse antes de recibir datos
- **Circuit Breaker**: Aislamiento automático de nodos fallidos
- **Ventanas Limitadas**: Prevención de acumulación ilimitada de datos
- **Max Connections**: Límite configurable de conexiones simultáneas
