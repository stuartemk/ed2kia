## 📋 Resumen

Sprint 5 introduce el sistema completo de **Dashboard v2 en tiempo real** con enrutamiento adaptativo y balanceo predictivo de carga:

1. **Dashboard UI v2** — Motor de estado y agregación de métricas con ventanas deslizantes, alertas umbral-configurables, rate limiting y snapshots con resumen de salud del sistema.
2. **WebSocket Dashboard Stream** — Streaming de datos del dashboard vía WebSocket con autenticación, suscripción por categoría, rate limiting por conexión y broadcast de alertas.
3. **Adaptive Router v2** — Enrutamiento adaptativo cross-model con balanceo predictivo, circuit breaker, reputación ponderada y fallback a Kademlia.
4. **Predictive Balancer** — Balanceo predictivo de carga basado en regresión lineal con clasificación de tendencias (increasing/stable/decreasing) y scoring compuesto por nodo.

## 📈 Estrategias de Enrutamiento

| Estrategia | Descripción |
|------------|-------------|
| **Adaptive** | Selección basada en puntuación compuesta (latencia + reputación + SLO) |
| **Predictive** | Selección basada en predicciones del balancer con umbral de confianza |
| **Kademlia Fallback** | Fallback a enrutamiento DHT cuando no hay nodos disponibles |

---

## 📈 Clasificación de Tendencias

| Tendencia | Condición | Acción |
|-----------|-----------|--------|
| **Increasing** | slope > threshold | Escalar o redirigir carga |
| **Stable** | abs(slope) <= threshold | Mantener carga actual |
| **Decreasing** | slope < -threshold | Aumentar carga al nodo |

## 📝 Migración

### De Dashboard v1 a v2

| v1 | v2 |
|----|----|
| `DashboardState` básico | Ventanas deslizantes con promedios |
| Sin alertas | Alertas por umbral con severidad |
| Sin rate limiting | Rate limiting configurable |
| Snapshot simple | Snapshot con resumen de salud |

### De Router v1 a v2

| v1 | v2 |
|----|----|
| Enrutamiento estático | Enrutamiento adaptativo |
| Sin circuit breaker | Circuit breaker con recovery |
| Sin reputación | Reputación ponderada con decay |
| Sin predictivo | Integración con Predictive Balancer |

---

## 🛡️ Consideraciones de Seguridad

- **Rate Limiting**: Protección contra abuso en todas las conexiones WebSocket
- **Autenticación**: Conexiones deben autenticarse antes de recibir datos
- **Circuit Breaker**: Aislamiento automático de nodos fallidos
- **Ventanas Deslizantes**: Prevención de acumulación ilimitada de métricas
