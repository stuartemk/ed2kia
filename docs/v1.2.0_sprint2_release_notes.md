## 📋 Resumen

Sprint 2 de v1.2.0 introduce **Cross-Chain Consensus, DAO Governance v3 & Distributed Fine-Tuning**:

1. **Cross-Chain Consensus** — Consenso basado en quórum con umbrales configurables, votación ponderada por reputación y time-lock para propuestas críticas.
2. **Bridge Validator** — Validación de transacciones cross-chain con pruebas Merkle, seguimiento de confirmaciones y máquina de estados para transacciones.
3. **DAO Governance v3** — Gobernanza descentralizada con votación híbrida (on-chain/off-chain), delegación de votos, umbrales de quórum y time-lock automático.
4. **Hybrid Voting Engine** — Motor de votación híbrida con detección de manipulación, pesos adaptativos por canal y ratio mínimo on-chain.
5. **Proposal Executor** — Ejecutor de propuestas con cola de prioridad (Critical/High/Normal/Low), time-lock y ledger de ejecución.
6. **Distributed Fine-Tuning** — Coordinación de fine-tuning distribuido con agregación FedAvg/Krum, detección de nodos stale y sincronización de gradientes.
7. **Checkpoint Manager** — Gestión de checkpoints de entrenamiento con políticas de retención, cadenas incrementales y restauración de gradientes.
8. **Fault Tolerance** — Tolerancia a fallos con circuit breaker (Closed/Open/HalfOpen), heartbeat de nodos, detección de nodos no saludables y estrategias de recuperación.

## 🔍 API Reference

### Cross-Chain Consensus

```rust
// Crear proof cross-chain
let proof = ChainProof::new(
    ProofType::MerkleInclusion,
    "eth".into(),   // source_chain
    "sol".into(),   // target_chain
    "proof_data".into(),
    "reference_hash".into(),
);

// Crear propuesta de consenso
let proposal = ConsensusProposal::new(
    "prop-1".into(),
    "eth".into(),
    "sol".into(),
    "Bridge upgrade".into(),
    vec![proof],
    false, // is_critical
);

// Votar
consensus.submit_vote("prop-1", "v1", VoteDirection::For, Some("reason".into()));
```

### Bridge Validator

```rust
// Registrar transacción
let tx = BridgeTransaction::new(
    "tx-1".into(),
    "eth".into(),
    "sol".into(),
    "sender".into(),
    "recipient".into(),
    1000,
    "ETH".into(),
);
validator.register_transaction(tx).unwrap();

// Validar con proof de lock
let proof = LockProof::new(
    "tx-1".into(),
    "eth".into(),
    "block_hash".into(),
    12345,
    vec!["merkle_branch".into()],
    "merkle_root".into(),
);
```

### DAO Governance v3

```rust
// Configurar DAO
let config = DaoConfig {
    quorum_threshold: 0.5,
    approval_threshold: 0.51,
    timelock_duration: Duration::from_secs(3600),
    voting_duration: Duration::from_secs(86400),
    min_proposal_stake: 10000,
    max_delegation_depth: 5,
    hybrid_onchain_ratio: 0.3,

// Crear propuesta
let proposal = DaoProposal::new(
    "prop-1".into(),
    "creator".into(),
    "Upgrade protocol".into(),
    "Description".into(),
    VoteType::OnChain,
    false, // is_critical
);

// Votar
dao.cast_vote("prop-1", "member", DaoVoteDirection::For, VoteType::OnChain, None);
```

### Distributed Fine-Tuning

```rust
// Configurar entrenamiento
let config = DistributedConfig {
    min_participants: 3,
    gradient_dim: 128,
    aggregation_method: AggregationMethod::FedAvg,
    max_epochs: 100,
    stale_threshold: Duration::from_secs(300),
};

// Registrar nodos
engine.register_node("node-1".into(), 1.0, 128).unwrap();

// Submit gradientes
let batch = GradientBatch::new("node-1".into(), 1, 0, gradients, 0.5);
engine.submit_gradient(batch).unwrap();

// Sincronizar
let summary = engine.sync_gradients().unwrap();
```

### Fault Tolerance

```rust
// Configurar tolerancia a fallos
let config = FaultToleranceConfig::new(5); // failure_threshold
let mut manager = FaultToleranceManager::new(config);

// Registrar nodos
manager.register_node("node-1".into()).unwrap();

// Verificar salud
let health = manager.get_node_health("node-1").unwrap();
let can_send = manager.can_send_to_node("node-1").unwrap();

// Detectar y recuperar
let recoveries = manager.detect_and_recover();
```

## 📊 Arquitectura

### Flujo de Consenso Cross-Chain

```
[Validator] → create_proposal() → [CrossChainConsensus]
                                    ↓
                              start_voting()
                                    ↓
[Validator] → submit_vote() × N → check_quorum_and_approval()
                                    ↓
                              quorum_reached → execute_proposal()
                                    ↓
                              [ConsensusResult]
```

### Flujo de Gobernanza DAO

```
[Member] → create_proposal() → [DaoGovernanceV3]
                                  ↓
                            start_voting()
                                  ↓
[Member] → cast_vote() × N → check_vote_thresholds()
                                  ↓
                        quorum_reached → check_timelock()
                                  ↓
                        timelock_expired → execute_proposal()
                                  ↓
                        [ExecutionResult]
```

### Flujo de Entrenamiento Distribuido

```
[Node] → register_node() → [DistributedFineTuning]
                               ↓
                         start_training()
                               ↓
[Node] → submit_gradient() × N → sync_gradients()
                               ↓
                    aggregate_gradients() (FedAvg/Krum)
                               ↓
                    [EpochSummary]
```

## 🔒 Seguridad

- **Circuit Breaker**: Protección contra fallos en cascada con estados Closed/Open/HalfOpen.
- **Time-Lock**: Propuestas críticas requieren período de espera antes de ejecución.
- **Detección de Manipulación**: Motor de votación híbrida detecta patrones de votación anómalos.
- **Quórum Configurables**: Umbrales de participación mínima para prevenir ataques Sybil.
- **Pruebas Merkle**: Validación criptográfica de transacciones cross-chain.
