## LP-64: Marketplace v3 & Cross-Chain Settlement

### Marketplace v3 (`src/marketplace/marketplace_v3.rs`)

Marketplace descentralizado de recursos con matching dinámico basado en SLO, reputación criptográfica y anti-gaming.

**Características:**
- Matching de órdenes con scoring ponderado (precio × reputación × SLO)
- Anti-gaming: Detección de actividad anómala por ventana temporal
- Gestión de listados con expiración automática
- Soporte multi-recurso: SAE, VRAM, Storage, Bandwidth, GPU
- Commitments criptográficos para settlement inmutable

**API Principal:**
```rust
let mut marketplace = MarketplaceV3::new();
marketplace.register_node("node-0".to_string(), 0.95, 1000.0);
let listing = ListingV3::new("listing-1", "node-0", ResourceKind::SAE, 0.5, 3600);
marketplace.list_resource(listing)?;
let matches = marketplace.match_orders();
```

### Escrow Ledger (`src/marketplace/escrow_ledger.rs`)

Ledger inmutable en `redb` con firmas `ed25519-dalek` para transacciones de escrow.

**Características:**
- Transacciones con firma criptográfica verificable
- Máquina de estados: Pending → Escrowed → Released/Refunded/Disputed
- Integración con ZKP para release condicional
- Auditoría completa con eventos inmutables
- SLO-aware: Release automático basado en métricas SLO

### Cross-Chain Settlement (`src/marketplace/cross_chain_settlement.rs`)

Motor de liquidación cross-chain con compromisos ark-bn254 y pruebas Merkle.

**Características:**
- Soporte para Ethereum, Solana, Polygon, Arbitrum
- Transiciones de estado validadas: Initiated → SourceConfirmed → Received → Finalized
- Timeouts automáticos con refund
- Balance tracking por chain con locking de fondos
- Pruebas Merkle para verificación de settlement

### Reputation Matcher (`src/marketplace/reputation_matcher.rs`)

Matching ponderado por reputación criptográfica con detección de Sybil.

**Características:**
- Scoring compuesto: reputación × cadena × SLO × latencia
- Detección de Sybil basada en patrones de IP
- Hash criptográfico de perfil para integridad
- Weighted matching con configuración de pesos ajustable

## LP-65: Alignment Loop v3 & Bias Mitigation

### Alignment Loop v3 (`src/alignment/loop_v3.rs`)

Loop de alineación con verificación ZKP y detección de sesgos en tiempo real.

**Características:**
- Feedback con prueba criptográfica (ZKP) para integridad
- Cálculo de confianza ponderado por historial
- Detección de bias score por ventana deslizante
- Fallback a modo estático cuando hay insuficiencia de datos
- Señales de steering con verificación ZKP

**API Principal:**
```rust
let mut loop_v3 = AlignmentLoopV3::new();
let feedback = FeedbackEntryV3::new("user-1", FeedbackTypeV3::Positive, 0.9, 0.95, "layer-1");
loop_v3.submit_feedback(feedback)?;
let result = loop_v3.run_cycle()?;
```

### Steering Verifier (`src/alignment/steering_verifier.rs`)

Verificador de señales de dirección con ZKP, validación de integridad y rate limiting.

**Características:**
- Verificación ZKP de señales de steering
- Validación de integridad criptográfica
- Rate limiting por fuente para prevenir spam
- Autorización de capas (layers)
- Tracking de estadísticas de verificación

### Bias Mitigator (`src/alignment/bias_mitigator.rs`)

Motor de detección y mitigación de sesgos en señales de alineación.

**Tipos de Sesgo Detectados:**
- **Skew**: Distribución sesgada de scores
- **Concentration**: Concentración de scores en rango estrecho
- **SourceDominance**: Dominio de fuente única
- **TemporalDrift**: Deriva temporal en scores
- **Quality**: Baja calidad general de datos

**Estrategias de Mitigación:**
- WeightAdjustment: Ajuste de pesos por detección
- SourceDownweight: Reducción de peso de fuentes dominantes
- SampleFiltering: Filtrado de muestras problemáticas
- LimitEnforcement: Aplicación de límites estrictos

### Gradient Aggregator v3 (`src/federation/gradient_aggregator_v3.rs`)

Agregación de gradientes FedAvg con tasas adaptativas y compresión.

**Características:**
- Agregación ponderada por reputación
- Detección de outliers con IQR
- Compresión de gradientes con ratio configurable
- Tracking de rondas con historial

## LP-66: Federation Scaling v3 & Sharding Adaptativo

### Federation Scaling v3 (`src/federation/scaling_v3.rs`)

Motor de escalado adaptativo de federación con scoring compuesto y decisiones automáticas.

**Características:**
- NodeCapabilityV3: Scoring compuesto (compute/memory/vram/bandwidth/load/latency/reputation)
- Decisiones: ScaleUp, ScaleDown, AddShard, RemoveShard, Rebalance, NoOp
- Asignación de nodos a shards con consistent hashing
- Detección de nodos stale por heartbeat
- Tracking de estadísticas completas

**API Principal:**
```rust
let mut scaling = FederationScalingV3::new();
let node = NodeCapabilityV3::new("node-0", 0.8, 0.8, 0.8, 0.8, 0.5, 50.0, 0.9, true);
scaling.register_node(node);
let decisions = scaling.evaluate();
```

### Adaptive Sharder (`src/federation/adaptive_sharder.rs`)

Gestión adaptativa de particiones con balanceo dinámico y migraciones.

**Características:**
- Creación/remoción dinámica de shards basada en load thresholds
- Routing de claves con consistent hashing
- Análisis de balance con acciones recomendadas (Split/Merge/Migrate)
- Gestión de migraciones con tracking de progreso
- Estados de shard: Active, Creating, Removing, Migrating, Paused

**API Principal:**
```rust
let mut sharder = AdaptiveSharder::new();
sharder.register_node("node-0", 0.5);
sharder.create_shard("shard-A", "node-0")?;
let shard = sharder.route_key("my-key");
let analysis = sharder.analyze_balance();
```

### Gradient Sync v3 (`src/federation/gradient_sync_v3.rs`)

Sincronización de gradientes con tolerancia a partición ≥99.5%.

**Características:**
- Sincronización con quorum configurable (default 67%)
- Detección de particiones de red
- Reconciliación post-partición
- Detección de divergencia vía cosine deviation
- Manejo de timeouts con estados de sync
- Tracking de reputación por nodo

**API Principal:**
```rust
let mut sync = GradientSyncV3::new();
sync.register_node("node-0");
let batch = sync.create_batch("node-0", vec![0.1, 0.2, 0.3], 1)?;
sync.sync_batch(&batch.batch_id, "node-1")?;
let quorum = sync.check_quorum(&batch.batch_id)?;
```
