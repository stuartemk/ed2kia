## LP-76: SAE Fine-Tuning v2

### Objetivo
Fine-tuning distribuido de SAEs con gradient compression y adaptive learning rates.

### Arquitectura
```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│  Node A      │    │  Node B      │    │  Node C      │
│  SAE Worker  │◄──►│  SAE Worker  │◄──►│  SAE Worker  │
│  Compressor  │    │  Compressor  │    │  Compressor  │
└──────┬───────┘    └──────┬───────┘    └──────┬───────┘
       │                   │                   │
       └───────────────────┼───────────────────┘
                           │
                  ┌────────▼────────┐
                  │  Coordinator    │
                  │  Aggregator v2  │
                  │  LR Scheduler   │
                  └─────────────────┘
```

### Entregables
- `src/sae/fine_tuning_v2.rs` - Engine de fine-tuning v2
- `src/sae/gradient_compressor.rs` - Compresión adaptativa de gradientes
- `src/sae/lr_scheduler.rs` - Adaptive learning rate scheduler
- Tests: ≥10 unit, ≥3 E2E

### Métricas
- Throughput: ≥125 samples/s (base: 100)
- Compression ratio: ≥4:1 (base: 2:1)
- Memory overhead: ≤10%

### APIs Clave
```rust
pub struct FineTuningV2Config {
    pub learning_rate: f64,
    pub compression_ratio: f32,
    pub batch_size: usize,
    pub adaptive_lr: bool,
}

pub struct FineTuningV2 {
    config: FineTuningV2Config,
    compressor: GradientCompressor,
    scheduler: LRScheduler,
}

impl FineTuningV2 {
    pub fn new(config: FineTuningV2Config) -> Self;
    pub fn train_step(&mut self, batch: &GradientBatch) -> Result<TrainingResult, FineTuningError>;
    pub fn adjust_learning_rate(&mut self, metric: f64) -> f64;
}
```

## LP-77: Cross-Node Compute Routing

### Objetivo
Routing inteligente de tareas de cómputo entre nodos con overhead ≤5%.

### Arquitectura
- **Load Predictor:** Predice carga de nodo basado en histórico
- **Router:** Selecciona nodo óptimo por tarea
- **Health Monitor:** Monitorea salud de nodos en tiempo real

### Entregables
- `src/scaling/compute_router.rs` - Router principal
- `src/scaling/load_predictor.rs` - Predictor de carga
- `src/scaling/health_monitor.rs` - Monitor de salud
- Tests: ≥10 unit, ≥3 E2E

### Métricas
- Routing overhead: ≤5%
- Prediction accuracy: ≥90%
- Failover time: ≤1s

### APIs Clave
```rust
pub struct ComputeRouter {
    predictor: LoadPredictor,
    monitor: HealthMonitor,
    nodes: HashMap<String, NodeStatus>,
}

impl ComputeRouter {
    pub fn route_task(&self, task: &ComputeTask) -> Result<String, RoutingError>;
    pub fn update_node_status(&mut self, node_id: &str, status: NodeStatus);
    pub fn get_optimal_node(&self, requirements: &ResourceRequirements) -> Option<String>;
}
```

## LP-79: Async ZKP v3

### Objetivo
Proofs ZKP asíncronos con latencia ≤150ms y batch accumulation.

### Arquitectura
- **Async Prover:** Generación de proofs en background
- **Batch Accumulator:** Acumulación de proofs para eficiencia
- **Verifier Pool:** Pool de verificadores concurrentes

### Entregables
- `src/zkp/async_prover_v3.rs` - Async prover v3
- `src/zkp/batch_accumulator_v3.rs` - Batch accumulator v3
- `src/zkp/verifier_pool_v3.rs` - Verifier pool v3
- Tests: ≥10 unit, ≥3 E2E

### Métricas
- Proof latency: ≤150ms (base: ~200ms)
- Batch throughput: ≥100 proofs/s
- Verification parallelism: ≥4x

### APIs Clave
```rust
pub struct AsyncZKPV3 {
    prover: AsyncProver,
    accumulator: BatchAccumulator,
    verifier_pool: VerifierPool,
}

impl AsyncZKPV3 {
    pub fn prove_async(&mut self, statement: Statement) -> ProofFuture;
    pub fn verify_batch(&self, proofs: &[Proof]) -> Vec<VerificationResult>;
    pub fn accumulate(&mut self, proof: Proof) -> Result<(), ZKPError>;
}
```
