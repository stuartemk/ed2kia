# Roadmap Técnico ed2kIA v1.3.0

## Visión General

v1.3.0 enfoca en mejorar la eficiencia del fine-tuning distribuido, optimizar el routing de cómputo entre nodos, fortalecer el ledger de reputación comunitaria y avanzar en proofs ZKP asíncronos de baja latencia.

**Línea base:** v1.2.0 STABLE
**Feature flag:** `stable` (consolidado)
**Objetivo:** Infraestructura más eficiente, accesible y verificable.

---

## Riesgos & Mitigación

| Riesgo | Impacto | Mitigación |
|--------|---------|------------|
| Regresión de rendimiento | Alto | Benchmarks automáticos en CI |
| Breaking changes en API | Medio | Versionado semántico, migration guides |
| Complejidad de ZKP v3 | Alto | Prototipo antes de producción |
| Dependencias externas | Medio | Audit de dependencias, pin versions |
| Fragmentación de red | Medio | Backward compatibility ≥2 versiones |

## Guardrails Técnicos

1. **Cero lógica financiera en código:** Reputación y staking son puramente técnicos
2. **Analogía Linux:** Infraestructura pública, incentivos técnicos
3. **Telemetría cero:** Código auditable, sin datos salientes
4. **Eficiencia energética:** Algoritmos optimizados para bajo consumo
5. **Acceso equitativo:** Sin barreras económicas para operar nodos
