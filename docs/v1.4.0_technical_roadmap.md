# ed2kIA v1.4.0 — Technical Roadmap

**Date:** 2026-05-10
**Base Version:** v1.3.0-stable
**Target:** v1.4.0-stable
**License:** Apache 2.0 + Ethical Use

---

## 1. Vision

v1.4.0 transitions ed2kIA from simulation-based prototypes to production-grade infrastructure. Key themes:
1. **Real Cryptographic Backends:** Replace hash-based proof simulation with actual zk-SNARK circuits (Halo2/Groth16).
2. **Production Concurrency:** Replace simulated parallel workers with `tokio`/`rayon` async runtime.
3. **Real Compression:** Integrate actual LZ4/ZSTD for checkpoint and gradient compression.
4. **Multi-Federation:** Support interoperability between independent ed2kIA federations.
5. **Observability:** Production-grade metrics, tracing, and alerting.

## 6. Guardrails (Inherited from v1.3.0)

- **Zero Financial Logic:** No tokens, no payments.
- **Zero Telemetry:** No external data collection.
- **Apache 2.0 + Ethical Use:** Mandatory license compliance.
- **Zero Unsafe:** Safe Rust only (except verified crypto crates).
- **Feature-Gated:** `cfg(feature = "stable")` for production code.

**Exception:** Real cryptographic backends (Halo2) may use `unsafe` internally, but ed2kIA code wrapping them must remain safe.
