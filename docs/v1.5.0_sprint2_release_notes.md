# v1.5.0 Sprint 2 Release Notes

## Overview

v1.5.0 Sprint 2 delivers public infrastructure, transparency, and technical governance capabilities across federation scaling, zero-knowledge proofs, cross-federation bridging, and real-time dashboards.

**Feature Flag:** `v1.5-sprint2`

## Modules Delivered

### LP-123: Federation Scaling v5 & Sharding Predictivo

- **Federation Scaling v5** (`src/federation/scaling_v5.rs`)
  - Cross-model federation scaling with reputation-weighted sharding
  - Partition tolerance with automatic detection
  - Predictive load forecasting with EMA smoothing
  - Accessor methods: `stats()`, `nodes()`, `shards()`

- **Predictive Sharder v5** (`src/federation/predictive_sharder_v5.rs`)
  - Load-based predictive sharding with cross-model awareness
  - Adaptive rebalancing with split/merge thresholds
  - Shard history tracking with EMA prediction
  - Accessor method: `histories()`

- **Gradient Sync v5** (`src/federation/gradient_sync_v5.rs`)
  - Distributed gradient synchronization with cross-model alignment
  - Adaptive compression (top-k)
  - EMA model registry with dimension validation
  - Accessor method: `models()`

### LP-124: Async ZKP v10 & Cross-Federation Bridge

- **Async ZKP v10** (`src/zkp/async_zkp_v10.rs`)
  - Adaptive proof scheduling with priority queue (BinaryHeap)
  - ML-based cost prediction per federation
  - Cross-federation delegation with credibility tracking
  - Replay protection with nonce set
  - Methods: `process_all()`, `get_federation()`, `proof_count()`

- **Federation ZKP Bridge v4** (`src/bridge/federation_zkp_bridge_v4.rs`)
  - Cross-federation proof verification with reputation routing
  - Merkle root aggregation for proof sessions
  - Consensus tracking with vote recording
  - Accessor methods: `metrics()`, `sessions()`, `nodes()`

### LP-125: UI Dashboard v6 & Real-time Streams

- **Dashboard v6** (`src/ui/dashboard_v6.rs`)
  - Unified state engine aggregating Scaling v5, ZKP v10 & Bridge v4
  - Automated alert generation for partition health, replay detection, assignment failures
  - Summary structs: `ScalingV5Summary`, `ZkpV10Summary`, `BridgeV4Summary`

- **WebSocket Federation Stream** (`src/web/ws_federation_stream.rs`)
  - Real-time event streaming for federation operations
  - Category-based subscriptions (Scaling, ZKP, Bridge, All)
  - Rate limiting per connection
  - Catch-up buffer with sequence tracking
