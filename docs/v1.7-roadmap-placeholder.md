## Vision

v1.7.0 will focus on **latency mitigation, performance scaling, and community growth** based on lessons learned from v1.6.0-stable deployment. The primary technical driver is [RFC-001: Latency Mitigation](rfc/rfc-001-latency-mitigation-v1.7.md), targeting a 7x reduction in tensor streaming round-trip latency (350ms → <50ms).

## Sprint Plan

### Sprint 1: RFC-001 PoC (Cuantización + Async Streaming)

**Duration:** 4 weeks
**Target:** FP8 quantization + async steering signals PoC

- [ ] Implement `src/sae/quantization.rs` (FP8 support)
- [ ] Implement `src/sae/async_steering.rs` (async signal queue)
- [ ] Integrate quantization hook in `src/bridge/tensor_flow.rs`
- [ ] Benchmark baseline vs FP8 in `benchmarks/benches/tensor_serialization.rs`
- [ ] Validate precision loss < 2% against test suite

**Acceptance Criteria:**
- Tensor streaming round-trip: < 200ms (from ~350ms)
- Precision loss (FP8): < 2%
- All 187 existing tests pass

### Sprint 2: Benchmark Suite + SIMD Optimizations

**Duration:** 4 weeks
**Target:** Complete benchmark suite + AVX2 optimizations

- [ ] Complete `benchmarks/benches/sae_loader.rs` with Candle integration
- [ ] Add FlatBuffers schema for quantized tensors
- [ ] Implement AVX2-optimized quantization paths
- [ ] Add CI benchmark tracking (GitHub Actions)
- [ ] Performance regression detection

**Acceptance Criteria:**
- Benchmark suite runs in CI
- SAE load (8192 latent): < 50ms
- FP8 serialization: > 500MB/s

### Sprint 3: Geographic Routing + libp2p RTT Integration

**Duration:** 4 weeks
**Target:** RTT-based federation routing

- [ ] Add RTT scoring to `src/bridge/federation_zkp_bridge_v7.rs`
- [ ] Geographic-aware shard assignment in `src/federation/scaling_v7.rs`
- [ ] RTT discovery protocol in `src/interop/protocol_adapter.rs`
- [ ] Fallback mechanism for overloaded nearby nodes

**Acceptance Criteria:**
- Network latency reduction: 30-50% (geographic)
- Zero precision loss
- Fallback < 100ms

### Sprint 4: Community Onboarding & First External PRs

**Duration:** 4 weeks
**Target:** First community contributions merged

- [ ] Create `good-first-issue` labels for performance track
- [ ] Publish community outreach materials ([`docs/community-outreach.md`](community-outreach.md))
- [ ] Onboard first 3 external contributors
- [ ] Review and merge first community PRs
- [ ] Document contribution workflow

**Acceptance Criteria:**
- ≥ 3 external PRs merged
- ≥ 5 active community contributors
- Contribution docs complete

## Community Input

### How to Contribute

1. **Propose:** Open issue with `[v1.7 proposal]` prefix
2. **Discuss:** Discord #roadmap channel
3. **Prototype:** Create feature branch + RFC doc
4. **Vote:** Governance proposal for inclusion

### Performance Track

- Run benchmarks: `cargo bench -p ed2kIA-benchmarks`
- Report results: Open issue with `performance` label
- Propose optimizations: SIMD, CUDA, serialization
- See: [`benchmarks/README.md`](../benchmarks/README.md)

### Feedback Channels

- GitHub Discussions: Weekly roadmap threads
- Discord: #roadmap, #dev-chat
- Governance: Formal proposals
- Surveys: Quarterly community priorities
